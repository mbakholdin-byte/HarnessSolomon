# Migration: v1.32 → v1.40

:::warning Breaking changes
This version introduces a new configuration layout and several API changes.
Read this guide carefully before upgrading.
:::

## Overview

Harness v1.40 consolidates scattered configuration files into a unified
`settings.yaml` with environment-variable overrides. The old `config.json`,
`providers.yaml`, and `.envrc` files are replaced.

## Key changes

- **Single settings file**: `settings.yaml` replaces `config.json` + `providers.yaml`
- **Environment variable prefix**: all Harness env vars now use `HARNESS_` prefix
- **Credentials moved**: API keys are no longer stored in config — use env vars or secrets manager
- **Memory backend**: SQLite → PostgreSQL migration is required for multi-user deployments
